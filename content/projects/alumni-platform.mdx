# Abstract

Educational institutions across India face a critical challenge in maintaining meaningful connections with their alumni networks. This project, developed for the **Smart India Hackathon 2023**, presents a comprehensive digital solution for centralized alumni data management, featuring role-based access control, geospatial analytics, and engagement tracking systems.

## 2. System Architecture

### 2.1 Technical Stack Selection

We employed a **modern 3-tier architecture** optimized for scalability and deployed on AWS infrastructure:

- **Frontend**: React.js with Redux Toolkit for state management
- **Backend**: Node.js/Express.js REST API with middleware architecture  
- **Database**: PostgreSQL with PostGIS extension for geospatial operations
- **Cloud Infrastructure**: AWS EC2, RDS, and S3 for file storage

### 2.2 Database Schema Design

To handle complex relationships between `Alumni`, `Batches`, `Events`, and `Donations`, we implemented a **normalized schema** following Third Normal Form (3NF):

```sql
-- Core Alumni Table with Geospatial Support
CREATE TABLE alumni (
    id SERIAL PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    graduation_year INT NOT NULL,
    degree_program VARCHAR(100) NOT NULL,
    current_company VARCHAR(255),
    current_position VARCHAR(255),
    location GEOMETRY(Point, 4326), -- PostGIS for map features
    linkedin_profile VARCHAR(500),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Event Management with Alumni Participation
CREATE TABLE events (
    id SERIAL PRIMARY KEY,
    title VARCHAR(255) NOT NULL,
    description TEXT,
    event_date TIMESTAMP NOT NULL,
    location_name VARCHAR(255),
    location_coords GEOMETRY(Point, 4326),
    max_participants INT,
    created_by UUID REFERENCES users(id)
);

-- Alumni-Event Relationship
CREATE TABLE event_participants (
    id SERIAL PRIMARY KEY,
    event_id INT REFERENCES events(id),
    alumni_id INT REFERENCES alumni(id),

    id SERIAL PRIMARY KEY,

    payment_method VARCHAR(50)

```

### 2.3 API Architecture

The **RESTful API** follows industry standards with proper HTTP status codes and consistent response formats:

```javascript
// Alumni Controller with Geospatial Queries
class AlumniController {
  // Get alumni within specific radius
  async getNearbyAlumni(req, res) {
    const { latitude, longitude, radius } = req.query;
    
    try {
      const query = `
        SELECT a.*, u.name, u.email,
               ST_Distance(
                 ST_GeogFromText('POINT(' || $2 || ' ' || $1 || ')'),
                 location
               ) as distance_meters
        FROM alumni a
        JOIN users u ON a.user_id = u.id
        WHERE ST_DWithin(
          location,
          ST_GeogFromText('POINT(' || $2 || ' ' || $1 || ')'),
          $3
        )
        ORDER BY distance_meters;
      `;
      
      const result = await pool.query(query, [latitude, longitude, radius]);
      
      res.json({
        success: true,
        data: result.rows,
        count: result.rowCount
      });
    } catch (error) {
      res.status(500).json({ success: false, error: error.message });
    }
  }

  // Advanced alumni search with filters
  async searchAlumni(req, res) {
    const { 
      graduation_year, 
      company, 
      degree_program,
      limit = 50,
      offset = 0 
    } = req.query;
    
    let whereConditions = [];
    let queryParams = [];
    let paramCount = 1;

    if (graduation_year) {
      whereConditions.push(`graduation_year = $${paramCount}`);
      queryParams.push(graduation_year);
      paramCount++;
    }
    
    if (company) {
      whereConditions.push(`current_company ILIKE $${paramCount}`);
      queryParams.push(`%${company}%`);
      paramCount++;
    }

    const whereClause = whereConditions.length > 0 
      ? `WHERE ${whereConditions.join(' AND ')}`
      : '';

    const query = `
      SELECT a.*, u.name, u.email
      FROM alumni a
      JOIN users u ON a.user_id = u.id
      ${whereClause}
      ORDER BY graduation_year DESC, u.name
      LIMIT $${paramCount} OFFSET $${paramCount + 1}
    `;

    queryParams.push(limit, offset);

    try {
      const result = await pool.query(query, queryParams);
      res.json({ success: true, data: result.rows });
    } catch (error) {
      res.status(500).json({ success: false, error: error.message });
    }
  }
}
```

## 3. Key Features Implementation

### 3.1 Role-Based Access Control (RBAC)

Security is paramount when handling alumni data. Our **RBAC system** implements fine-grained permissions:

```javascript
// Middleware for role-based authorization
const authorize = (allowedRoles) => {
  return async (req, res, next) => {
    try {
      const token = req.headers.authorization?.split(' ')[1];
      const decoded = jwt.verify(token, process.env.JWT_SECRET);
      
      const user = await User.findById(decoded.userId).populate('role');
      
      if (!user || !allowedRoles.includes(user.role.name)) {
        return res.status(403).json({
          success: false,
          message: 'Insufficient permissions'
        });
      }
      
      req.user = user;
      next();
    } catch (error) {
      res.status(401).json({ success: false, message: 'Invalid token' });
    }
  };
};

// Usage in routes
router.get('/alumni', 
  authorize(['admin', 'faculty']), 
  AlumniController.getAllAlumni
);

router.get('/alumni/:id/contact',
  authorize(['admin']), // Only admins can access contact info
  AlumniController.getContactInfo
);
```

## 6. Technical Challenges and Solutions

### 6.1 Scalability Challenges

**Challenge**: Handling large datasets (50,000+ alumni) with real-time geospatial queries.

**Solution**: Implemented **database sharding** by graduation year and **Redis caching** for frequently accessed data:

```javascript
// Redis Cache Implementation
class CacheService {
  static async getAlumniByLocation(lat, lng, radius) {
    const cacheKey = `alumni:${lat}:${lng}:${radius}`;
    
    // Try cache first
    const cached = await redis.get(cacheKey);
    if (cached) {
      return JSON.parse(cached);
    }
    
    // Fallback to database
    const alumni = await AlumniModel.findNearby(lat, lng, radius);
    
    // Cache for 5 minutes
    await redis.setex(cacheKey, 300, JSON.stringify(alumni));
    
    return alumni;
  }
}
```

### 6.2 Data Privacy and Security

**Challenge**: Protecting sensitive alumni information while enabling discovery features.

**Solution**: Implemented **tiered access levels** and **data anonymization**:

```javascript
// Privacy-aware alumni data serialization
class AlumniSerializer {
  static serialize(alumni, userRole) {
    const baseData = {
      id: alumni.id,
      name: alumni.name,
      graduation_year: alumni.graduation_year,
      degree_program: alumni.degree_program
    };

    switch (userRole) {
      case 'admin':
        return {
          ...baseData,
          email: alumni.email,
          phone: alumni.phone,
          current_company: alumni.current_company,
          location: alumni.location
        };
      
      case 'faculty':
        return {
          ...baseData,
          current_company: alumni.current_company,
          location: alumni.location // Approximate location only
        };
      
      case 'student':
        return {
          ...baseData,
          current_company: alumni.current_company
          // No contact info or precise location
        };
      
      default:
        return baseData;
    }
  }
}
```
